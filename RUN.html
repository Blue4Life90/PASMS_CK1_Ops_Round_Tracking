Running Your Refactored Operator Rounds Application

To run your newly refactored Operator Rounds Tracking application, follow these steps:

Step 1: Organizing Your Project Files

Ensure your project directory is structured as follows:

operator_rounds/
├── __init__.py
├── app.py
├── config.py
├── database/
│   ├── __init__.py
│   ├── connection.py
│   ├── schema.py
│   ├── queries.py
│   └── models.py
├── ui/
│   ├── __init__.py
│   ├── sidebar.py
│   ├── round_completion.py
│   ├── section_editor.py
│   ├── view_rounds.py
│   └── forms.py
└── utils/
    ├── __init__.py
    ├── validation.py
    ├── state.py
    ├── export.py
    └── helpers.py

Step 2: Creating a Setup File

Create a setup.py file in the parent directory:

from setuptools import setup, find_packages

setup(
    name="operator_rounds",
    version="1.0.0",
    packages=find_packages(),
    install_requires=[
        "streamlit>=1.20.0",
        "pandas>=1.3.0",
        "sqlite3",
    ],
)

Step 3: Installing Your Package

Run the following command in your terminal:

pip install -e .

Step 4: Running the Application

To start your Streamlit application, run:

streamlit run operator_rounds/app.py

Step 5: Verifying Your Application

Ensure that everything is working correctly:

  • Select a round type
  • Enter operator information
  • Add sections and items
  • Complete rounds
  • View previous rounds

Troubleshooting Common Issues

Module Import Errors

Problem: ModuleNotFoundError: No module named 'operator_rounds'

Solution: Ensure you've installed the package and restart your terminal.

Database File Location

Problem: Database file not found

Solution: Modify config.py to specify the database path.

Streamlit State Management

Problem: UI components not updating

Solution: Use st.write(st.session_state) to debug.

Advanced Running Options

# Run on a specific port
streamlit run operator_rounds/app.py --server.port 8501

# Allow external connections
streamlit run operator_rounds/app.py --server.address 0.0.0.0

# Increase memory limit
streamlit run operator_rounds/app.py --server.maxUploadSize 200

Development Workflow

Follow these steps when making changes:

  1. Edit the relevant file(s)
  2. Save changes
  3. Streamlit will auto-reload the UI
  4. Refresh manually if needed

Creating a Production Deployment

Before deploying, ensure:

  • Debug toggles are removed
  • Environment variables are used for sensitive configurations
  • Database backups are in place
  • Deployment via Streamlit sharing, Docker, or hosting platforms

By following these steps, you should have a fully functional application.
